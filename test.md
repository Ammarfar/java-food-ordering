## Take-Home Assignment: Full Stack Food Ordering Application

### Objective

The goal of this assignment is to evaluate your ability to build a clean, functional full-stack application while demonstrating good coding practices, basic architecture, and clear documentation.

---

### Technical Requirements

#### Backend

- Use **Java with Spring Boot** to build a RESTful API.
- Use **MySQL** as the relational database.
- Follow a clean structure (Controller → Service → Repository).
- Include **unit tests** for core business logic.

#### Frontend

- Build a **responsive web application** using **Angular**.
- Ensure usability on both desktop and mobile screen sizes.
- Integrate with your backend API.

---

### Core Features (Minimum Scope)

#### User Role

- View list of food items
- View food item details
- Add items to cart
- Place an order

#### Admin Role

- Create, update, delete food items
- View list of orders

---

### Notes on Scope

- You are **not required to implement authentication/login**.
  You may simulate roles (User/Admin) using simple logic (e.g., toggle, separate views, or hardcoded role).
- Focus on **functionality, structure, and clarity** over completeness.

---

### Bonus Features (Optional – for Extra Credit)

You may implement any of the following to showcase your skills:

#### Backend

- Basic authentication (e.g., JWT-based login)
- Pagination and filtering for menu/orders
- Order status management (e.g., Pending, Completed)
- Integration tests in addition to unit tests

#### Frontend

- Better UI/UX (loading states, error handling, clean layout)
- State management (e.g., Angular services or NgRx)
- Form validation and user feedback

#### General

- API documentation using Swagger/OpenAPI
